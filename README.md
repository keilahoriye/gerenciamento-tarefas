# Gerenciador de Tarefas
O projeto consiste em um Sistema de Gerenciamento de Tarefas desenvolvido como exercício prático para a disciplina de Engenharia de Software. O foco é a aplicação de Scrum, Kanban e integração contínua com GitHub Actions.

## Tecnologias Utilizadas
* Linguagem: Python
* Interface: Linha de Comando
* Gestão: GitHub Projects e Issues
* CI/CD: GitHub Actions
* Versionamento: Git e GitHub

## Funcionalidades do Sistema
O sistema permite a gestão de atividades através das seguintes operações:
- Criar tarefa
- Editar tarefa
- Excluir tarefa
- Listar tarefas
- Marcar tarefa como concluída

## Metodologia e Organização
### 1. Scrum e Product Backlog
O planejamento foi estruturado em uma Sprint única. As funcionalidades foram definidas como User Stories no Backlog do produto.

### 2. Kanban (GitHub Projects)
O fluxo de trabalho foi gerenciado no GitHub Projects através das colunas:
* To Do: Tarefas aguardando início.
* In Progress: Tarefas em desenvolvimento.
* Review: Código em fase de revisão.
* Done: Funcionalidades finalizadas.

### 3. Gestão de Issues
Cada tarefa possui uma Issue dedicada com:
* Labels;
* Responsáveis atribuídos;
* Vínculo direto com o projeto Kanban.

## DevOps e Automação (GitHub Actions)
Foi configurado um workflow no GitHub Actions para garantir a integração contínua (CI).
